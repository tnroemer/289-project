# Lesion-Focused Classification for Skin Cancer Detection

**Does background suppression help generalization?**
CS 289A, Spring 2026 — Tobias Roemer, Thomas Lee, Leo Li.

Skin-lesion classifiers trained end-to-end on dermoscopic images can latch onto
non-lesion context (skin texture, hair, rulers, lighting) instead of the lesion
itself. This project tests whether **explicitly localizing the lesion before
classification** improves performance and out-of-distribution generalization,
using HAM10000 for in-distribution training and PAD-UFES-20 as an
out-of-distribution clinical-photograph test set.

## What's here

- [`notebooks/01_segmentation.ipynb`](notebooks/01_segmentation.ipynb) — trains
  the from-scratch U-Net lesion segmenter and evaluates it (Dice / threshold
  sweep) on ISIC-2018-style data.
- [`notebooks/02_lesion_focused.ipynb`](notebooks/02_lesion_focused.ipynb) —
  loads the trained segmenter and turns raw images into background-suppressed
  *lesion-focused* images.
- `src/` — all real logic. `src/segmentation_unet.py` backs both notebooks; the
  full HAM10000/PAD-UFES-20 training & evaluation pipeline lives in
  `src/{data_setup,training,evaluation}` (see the
  [README](https://github.com/tnroemer/289-project#readme)).
- `report/main.tex` — the written report.

## Running the notebooks

Both notebooks default to `SAMPLE = True` and run on a small **synthetic**
dataset (generated by `python -m data_setup.make_sample_data`), so they execute
in seconds on CPU with no kagglehub credentials — this is what the Binder
launcher and this site use. Set `SAMPLE = False` to point them at the real
HAM10000 / ISIC-2018 data on a machine that has it.
